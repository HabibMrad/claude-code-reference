Claude Code is an agentic coding engine — not a chat window. It reads your entire codebase, edits files, runs commands, and coordinates sub-agents, all without upload limits or session expiry. Available in Terminal, VS Code, JetBrains, Desktop app, Web, and iOS.

📦 Installation
# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash

# Windows PowerShell
irm https://claude.ai/install.ps1 | iex

# Windows CMD
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd

# Homebrew (stable, ~1 week behind)
brew install --cask claude-code

# WinGet
winget install Anthropic.ClaudeCode

# Start anywhere
cd your-project && claude

Model Context Protocol (MCP)

MCP is USB-C for AI: one open standard, hundreds of integrations. Claude Code connects to any MCP server to read design docs, update tickets, query databases, and more.
➕ Adding MCP Servers
# HTTP (recommended for remote)
claude mcp add --transport http notion \
  https://mcp.notion.com/mcp

# SSE (legacy, use HTTP instead)
claude mcp add --transport sse asana \
  https://mcp.asana.com/sse

# stdio (local process)
claude mcp add --transport stdio \
  --env AIRTABLE_API_KEY=YOUR_KEY airtable \
  -- npx -y airtable-mcp-server

# With auth header
claude mcp add --transport http my-api \
  https://api.example.com/mcp \
  --header "Authorization: Bearer token"

# Windows: wrap with cmd /c
claude mcp add --transport stdio myserver \
  -- cmd /c npx -y @some/package
📋 Managing Servers
# List all servers
claude mcp list

# Get details for a server
claude mcp get github

# Remove a server
claude mcp remove github

# Check server status (interactive)
/mcp

🧠 Auto Memory
# Auto-memory location
~/.claude/projects/<proj>/memory/

MEMORY.md       # loads at startup
                # (first 25KB or 200 lines)
                # topic files load on demand

Claude automatically builds memory as it works — saving build commands, debugging insights, project patterns — without you writing anything.

Toggle or edit via /memory. Disable via CLAUDE_CODE_DISABLE_AUTO_MEMORY=1.

Workflows & Best Practices

🗺️ Plan Mode Workflow
Shift+Tab
Cycle: Normal → Auto-Accept → Plan mode
/plan [desc]
Jump directly into plan mode
--permission-mode plan
Start CLI session in plan mode

Plan files are now named after your prompt, e.g. fix-auth-race-snug-otter.md

🔁 Checkpoint + Iterate Pattern
1. Ask to plan
Let Claude propose approach before coding
2. Review plan
Edit plan file, approve or modify
3. Let it execute
Claude implements + creates checkpoints
4. Esc Esc to rewind
If something goes wrong
5. Iterate
Give specific feedback, repeat
🌿 Git Worktrees (/batch)
# Parallel work on multiple branches
/batch migrate all API endpoints to v2

# CLI flag for isolated branch
claude --worktree feature-name "..."

# Agent in its own worktree
isolation: worktree   # in agent frontmatter

worktree.sparsePaths: ["src/auth/**"]
